    • Defined in TransactionController.ts:191

    Add a new unapproved transaction to state. Parameters will be validated, a unique transaction id will be generated, and gas and gasPrice will be calculated if not provided. If A <tx.id>:unapproved hub event will be emitted once added.

    Parameters

    • transaction: Transaction

      Transaction object to add

    • Optional origin: undefined | string

      Domain origin to append to the generated TransactionMeta

    Returns Promise<Result>

    • Object containing a promise resolving to the transaction hash if approved

approveTransaction

  • approveTransaction(transactionID: string): Promise<void>
    • Defined in TransactionController.ts:243

    Approves a transaction and updates it's status in state. If this is not a retry transaction, a nonce will be generated. The transaction is signed using the sign configuration property, then published to the blockchain. A <tx.id>:finished hub event is fired after success or failure.

    Parameters

    • transactionID: string

      ID of the transaction to approve

    Returns Promise<void>

    • Promise resolving when this operation completes
